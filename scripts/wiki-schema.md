# Wiki Schema

Defines the structure, frontmatter fields, and update-vs-create logic for
AI-Vault library notes generated by `generate-book-moc.py`.

## Note Frontmatter Fields

| Field | Type | Description | Set by |
|-------|------|-------------|--------|
| `title` | string | Book or paper title | Pipeline (from metadata or filename) |
| `author` | string | Author name(s) | Pipeline |
| `category` | string | Library category (e.g. "Books/Cybersec") | Pipeline |
| `file_type` | string | "pdf" or "epub" | Pipeline |
| `usb_path` | string | Library-relative path to source file | Pipeline |
| `pages` | int | Page count (PDFs only) | Pipeline |
| `imported` | ISO date | Date the note was first created | Pipeline |
| `tags` | list | Always includes "library"; also the category slug | Pipeline |
| `cross_refs` | list | Wiki-links to related notes in AI-Vault | Claude (via obsidian_patch) |
| `summary_status` | string | "pending" or "done" | Pipeline (initial), Claude (on summarize) |

## Note Sections

Every generated note contains these sections in order:

1. Frontmatter (YAML block)
2. Title heading (`# Title`)
3. Author / Type / USB path metadata block
4. `## Summary` -- placeholder until Claude fills it in via `obsidian_patch`
5. `## Table of Contents` -- extracted from source, no AI

## Update vs. Create Logic

**Create** a new note when: the source file has not been processed before
(`dst.exists()` is False).

**Update** an existing note when: the source file already has a note AND:
- The `summary_status` is "pending" (summary not yet generated)
- The user explicitly requests a re-index
- The source file has changed (future: checksum-based detection)

Do NOT overwrite existing notes silently. If a note exists with `summary_status: "done"`,
treat it as read-only unless the user explicitly requests an update.

## Cross-Reference Guidelines

`cross_refs` is populated by Claude (not by the pipeline) when:
- Summarizing a book and noticing thematic overlap with another library note
- The user asks Claude to "link related books"
- Running a periodic cross-reference pass across the library

Format: YAML list of Obsidian wiki-link strings:
```yaml
cross_refs:
  - "[[Clean Code - Robert Martin]]"
  - "[[The Pragmatic Programmer]]"
```

## log.md Format

`<vault-root>/02. AI-Vault/Library/log.md` records pipeline activity:

```
# Library Log

[YYYY-MM-DD] Created [[Title]] (Category)
[YYYY-MM-DD] Updated [[Title]] -- summary added
```

Entries are append-only. Never truncate log.md.

## index.md (future)

`<vault-root>/02. AI-Vault/Library/index.md` -- a browsable overview of all library
notes, organized by category with summary status indicators. Generated by
`generate-library-moc.py` (already exists). Reserved for future enrichment:
add summary status counts per category, last-updated timestamp.
