## [2026-03-27] ADR-005: Config Protection Hook

Status: ACTIVE

Context: ECC research identified config-protection as high-priority hook. PSC CLAUDE.md forbids debug output but doesn't protect linter configs that enforce this rule. Without protection, Claude may weaken config strictness to bypass lint errors rather than fixing the underlying code issues.

Decision: Implement pre:config-protection.sh hook blocking writes to 43 linter/formatter config file patterns (ESLint, Prettier, Biome, Ruff, ShellCheck, Pylint, MyPy, StyleLint, RuboCop, golangci-lint, Clippy). Exception: pyproject.toml allowed (contains dependencies and project metadata, not just lint config).

Rationale: Prevents accidental weakening of code quality standards. Complements existing block-secrets-write.sh by covering different risk domain. Low false positive rate due to pyproject.toml exemption. Exit 2 (block) requires manual override if modification is intentional.

Consequences: Claude cannot modify protected config files via Write/Edit tools. Manual file editing or hook modification required for legitimate config changes. Encourages fixing code to meet standards rather than lowering standards to accept code.

Confidence: High

## [2026-03-27] ADR-004: Cost Tracking Hook Limitation

Status: BLOCKED

Context: ECC includes stop:cost-tracker hook that logs token counts and estimated costs per session. This feature would provide visibility into API usage costs and help identify expensive operations for optimization.

Decision: Implement stub hook (stop-cost-tracker.sh) that logs "NOT_IMPLEMENTED" status. Document requirement for upstream Claude Code enhancement to expose token/cost data in Stop hook input.

Rationale: Stop hooks in Claude Code currently receive only {session_id, stop_reason, os, arch} in stdin. Token counts, model name, and API response metadata are not available. Alternative approaches were evaluated and rejected: CLI log parsing (undocumented format, brittle), MCP server integration (requires external infrastructure), estimation heuristics (inaccurate, misleading). The correct solution requires upstream protocol enhancement.

Consequences: Cost tracking feature remains non-functional until Claude Code adds required fields to Stop hook input. Stub reserves telemetry file (context/telemetry/costs.jsonl) and documents the blocker. No workaround available without upstream support.

Required Enhancement: Stop hooks need additional JSON fields in stdin: `model` (string, e.g. "claude-sonnet-4-5-20250929"), `usage.input_tokens` (integer), `usage.output_tokens` (integer), `usage.cache_read_tokens` (integer, optional), `usage.cache_creation_tokens` (integer, optional).

Constraint: Upstream Claude Code hook protocol

Alternative: Deferred MCP-based cost tracking to Phase 2


## [2026-03-31] ADR-007: Agent Memory Scoping

Status: ACTIVE

Context: Claude Code agents inherit the invoking session's memory context by default. For psc_comet, project-specific context (architecture, constraints, active decisions) should be available to subagents without requiring them to load user-scoped global memory.

Decision: Add `memory_scope: project` to all 13 agent frontmatters. Scaffold `.claude/agent-memory/MEMORY.md` as the project-scoped memory index for agents.

Rationale: Separating user memory (preferences, cross-project patterns) from project memory (architecture, constraints) prevents agents from acting on user-specific context that is irrelevant to a given project. Reduces noise in agent context.

Consequences: Agents read `.claude/agent-memory/MEMORY.md` for project context. User preferences remain in `~/.claude/memory/`. Index maintained by the distill skill.

Confidence: Medium
