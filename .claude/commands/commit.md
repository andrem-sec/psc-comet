Invoke the git-commit skill now. Review the staged changes. Write a conventional commit message (type(scope): description) with a body explaining why, not what. For any non-trivial decision: add Constraint:, Rejected:, Directive:, or Confidence: trailers as applicable. Subject line under 72 characters. Imperative mood.
