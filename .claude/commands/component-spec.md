name: component-spec
description: Define component API, states, variants, and ARIA requirements before writing any implementation code
---

Invoke the component-spec skill now. For the target component: identify its category (input, display, navigation, feedback, layout, overlay). Define all props with types and defaults. Enumerate every visual state (default, hover, focus, active, disabled, loading, error, empty). List named variants as prop values. Specify ARIA role, required attributes, and keyboard navigation. Define responsive behavior at each breakpoint. Present the full spec and wait for explicit confirmation before writing any implementation code.
