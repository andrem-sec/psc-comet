---

Invoke the plan-first skill now. Assess scope (file count, domains, security sensitivity). If 3+ files, cross-domain, or security-sensitive: produce a full plan in the standard format and wait for confirmation before any edits begin. If scope is small enough to proceed directly, say so explicitly and state what you are about to do.

For each phase in the plan, include:

**Files in scope** — explicit list of every file that will be touched in this phase. Nothing outside this list should be edited without re-planning.

**Files off-limits** — explicit list of files adjacent to the work that must not be modified in this phase.

**Known pitfalls** — specific traps, gotchas, or failure modes to watch for in this phase, drawn from the codebase, prior learnings, or the nature of the change.

These three sections are mandatory for every phase. A plan without them is incomplete.
