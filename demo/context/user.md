## Role

Senior backend engineer. Primary language: TypeScript (Node.js). Secondary: Python for tooling and scripts.

## Working Style

- Prefer understanding the why before implementing — ask clarifying questions early, not mid-implementation
- Code review after every non-trivial change, not just before merge
- TDD for business logic; not strictly required for infrastructure/config
- Short sessions (1-2 hours) with clear goals; prefer to finish a feature than leave it half-done

## Preferences

- Conventional commits — no emoji in commit messages
- No debug logging in production code
- Type everything explicitly — no implicit any
- Tests alongside code, not in a separate PR
- Keep PRs focused — one logical change per PR

## What I Find Annoying

- Claude starting to implement before the scope is clear
- Summaries of what just happened ("I have now completed...") — just show the diff
- Suggestions to refactor things that weren't part of the task
- Long explanations when a short one would do
